# 1523. Count Odd Numbers in an Interval Range

Given two non-negative integers `low` and `high`. 
Return the count of odd numbers between `low` and `high` (inclusive).

**Constraints:**

- `0 <= low <= high <= 10^9`

## 基礎思路

本題要求計算區間 `[low, high]` 中的奇數數量。
由於數值範圍可能高達 `10^9`，不適合逐一檢查每個整數，因此需使用數學方式直接推導奇數的個數。

可掌握以下核心觀察：

* **奇數的分佈具有規律性**：在非負整數序列中，每隔一個數就會出現一個奇數，因此可透過整體計數方式快速取得特定範圍內的奇數個數。

* **計算某個數 `x`（含 0）之前有多少奇數，可使用 `(x + 1) // 2`**
  因為奇數為：1、3、5、…，其計數規律可直接由除 2 取得。

* **區間 `[low, high]` 的奇數數量可由「奇數個數前綴」相減而得**
  只要計算 `0..high` 的奇數數量，再減去 `0..(low-1)` 的奇數數量，即可得到區間內奇數的個數。

* **右移運算可實現高效整數除 2**
  在 JavaScript 中使用無號右移可保證以整數方式運作，避免浮點問題。

基於這些觀察，可將計算直接化為兩次奇數計數的差值，達到常數時間解法。

## 解題步驟

### Step 1：直接以兩個前綴奇數計數相減得到答案

透過 `(high + 1) >>> 1` 取得 `0..high` 區間中的奇數個數，再透過 `(low >>> 1)` 取得 `0..(low-1)` 的奇數個數，兩者相減即為 `[low, high]` 的奇數個數。

```typescript
// 計算至 high 與至 low-1 的奇數個數差，即為 [low, high] 的奇數數量
return ((high + 1) >>> 1) - (low >>> 1);
```

## 時間複雜度

- 僅使用常數次計算與位運算。
- 未進行迴圈或額外的條件判斷。
- 總時間複雜度為 $O(1)$。

> $O(1)$

## 空間複雜度

- 未使用任何額外陣列。
- 僅使用常數空間。
- 總空間複雜度為 $O(1)$。

> $O(1)$
