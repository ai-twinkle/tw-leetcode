# 3474. Lexicographically Smallest Generated String

You are given two strings, `str1` and `str2`, of lengths `n` and `m`, respectively.

A string word of length `n + m - 1` is defined to be generated by `str1` and `str2` 
if it satisfies the following conditions for each index `0 <= i <= n - 1`:

- If `str1[i] == 'T'`, the substring of `word` with size `m` starting at index `i` is equal to `str2`, 
  i.e., `word[i..(i + m - 1)] == str2`.
- If `str1[i] == 'F'`, the substring of `word` with size `m` starting at index `i` is not equal to `str2`, 
  i.e., `word[i..(i + m - 1)] != str2`.

Return the lexicographically smallest possible string that can be generated by `str1` and `str2`. 
If no string can be generated, return an empty string `""`.

**Constraints:**

- `1 <= n == str1.length <= 10^4`
- `1 <= m == str2.length <= 500`
- `str1` consists only of `'T'` or `'F'`.
- `str2` consists only of lowercase English characters.

## 基礎思路

本題要求構造一個長度為 `n + m - 1` 的字串，使其同時滿足所有 `T` 位置與 `F` 位置的約束，並在合法解中選取字典序最小的結果。問題的核心在於兩類約束之間的衝突處理：`T` 約束要求某段子串必須與 `str2` 完全相同，而 `F` 約束則要求某段子串必須在至少一個位置與 `str2` 不同。

在思考解法時，可掌握以下核心觀察：

- **`T` 約束是強制性的**：
  每個 `T` 位置會對輸出字串中的一段連續區間施加確定性要求，所有 `T` 約束必須優先處理，並偵測衝突——若兩個 `T` 窗口對同一位置要求不同字元，則無解。

- **`F` 約束可被延後驗證**：
  先完成所有 `T` 約束的寫入後，再逐一檢查每個 `F` 窗口是否已自然與 `str2` 產生差異；若整個窗口完全吻合，才需要主動修改。

- **字典序最小的修改策略**：
  若某個 `F` 窗口需要強制製造差異，應選擇最靠右的自由位置（未被 `T` 鎖定的位置）進行修改，並將其設為字典序最小的合法字元（與 `str2` 對應字元不同的最小字元）。選擇最靠右的位置能避免影響左側字元，從而盡可能保持字典序的最小性。

- **無解的兩種情形**：
  一是兩個 `T` 窗口在同一位置要求不同字元；二是某個 `F` 窗口的所有位置均被 `T` 鎖定且完全吻合 `str2`，無任何自由位置可修改。

依據以上特性，可以採用以下策略：

- **先以 `'a'` 填滿整個輸出陣列**，確保在未受任何約束時字典序最小。
- **優先套用所有 `T` 約束**，將對應位置蓋寫為 `str2` 的字元，並標記為鎖定。
- **逐一驗證 `F` 約束**，若窗口已自然不同則跳過；若完全吻合，在最靠右的自由位置填入最小的差異字元。

此策略確保輸出在所有合法解中字典序最小，同時能正確偵測並回報無解情形。

## 解題步驟

### Step 1：初始化輸出陣列與輔助結構

建立長度為 `n + m - 1` 的輸出陣列，以 `'a'`（ASCII 97）填滿作為最小基底；同時建立鎖定標記陣列，以及預先計算 `str2` 各字元的字元碼，避免重複轉換。

```typescript
const n = str1.length;
const m = str2.length;
const totalLength = n + m - 1;

const result = new Uint8Array(totalLength);

// 以字典序最小的基底字元 'a' 填滿
result.fill(97);

// 記錄被 T 約束鎖定的位置
const lockedByT = new Uint8Array(totalLength);

// 預先計算 str2 的字元碼以加快存取
const str2Codes = new Uint8Array(m);
for (let j = 0; j < m; j++) {
  str2Codes[j] = str2.charCodeAt(j);
}
```

### Step 2：套用所有 T 約束並偵測衝突

遍歷 `str1` 中每個 `'T'` 位置，將對應窗口內的每個位置蓋寫為 `str2` 的字元，並標記為鎖定。若某位置已被鎖定且要求的字元與目前值不同，代表兩個 `T` 窗口在此位置產生衝突，直接回傳空字串。

```typescript
// 優先套用所有 T 約束：將 str2 印入每個 T 窗口
for (let i = 0; i < n; i++) {
  if (str1[i] !== 'T') {
    continue;
  }

  for (let j = 0; j < m; j++) {
    const position = i + j;
    const requiredCode = str2Codes[j];

    if (lockedByT[position] && result[position] !== requiredCode) {
      // 兩個 T 窗口在此位置要求不同字元
      return "";
    }

    result[position] = requiredCode;
    lockedByT[position] = 1;
  }
}
```

### Step 3：驗證 F 約束並在必要時製造差異

遍歷 `str1` 中每個 `'F'` 位置，掃描其對應窗口，同時尋找最靠右的自由位置（未被 T 鎖定）以及判斷窗口是否已自然與 `str2` 有所不同。若窗口已自然產生差異，則無需任何修改，直接跳過。

```typescript
// 驗證 F 約束：每個 F 窗口必須在某個位置與 str2 不同
for (let i = 0; i < n; i++) {
  if (str1[i] !== 'F') {
    continue;
  }

  // 尋找最靠右的自由位置，同時確認窗口是否已自然不同
  let rightmostFreePosition = -1;
  let alreadyDiffers = false;

  for (let j = 0; j < m; j++) {
    const position = i + j;

    if (!lockedByT[position]) {
      rightmostFreePosition = j;
    }

    if (result[position] !== str2Codes[j]) {
      alreadyDiffers = true;
    }
  }

  if (alreadyDiffers) {
    continue;
  }

  // ...
}
```

### Step 4：強制在最靠右的自由位置填入最小差異字元

當窗口完全吻合 `str2` 且必須強制製造差異時，若不存在任何自由位置，代表所有位置均被 `T` 鎖定且完全吻合，此為無解情形，回傳空字串。否則，在最靠右的自由位置填入字典序最小的合法字元：若 `str2` 對應字元非 `'a'`，則填 `'a'`；否則填 `'b'`。

```typescript
for (let i = 0; i < n; i++) {
  // Step 3：掃描窗口尋找自由位置並判斷是否已自然不同

  // 窗口完全吻合 str2，必須主動破壞一致性
  if (rightmostFreePosition === -1) {
    // 所有位置均被 T 鎖定且吻合 str2，無法修改
    return "";
  }

  // 在最靠右的自由位置填入與 str2 不同的最小字元
  const flipPosition = i + rightmostFreePosition;
  const conflictCode = str2Codes[rightmostFreePosition];

  // 若 str2 對應字元非 'a' 則填 'a'，否則填 'b'
  result[flipPosition] = conflictCode !== 97 ? 97 : 98;
}
```

### Step 5：將結果陣列轉為字串並回傳

所有約束均已處理完畢，將 `Uint8Array` 中的字元碼轉換為字串並回傳。

```typescript
return String.fromCharCode(...result);
```

## 時間複雜度

- 預先計算 `str2` 字元碼需 $O(m)$；
- 套用 T 約束時，每個 `'T'` 位置展開 $O(m)$ 的窗口，最多 $n$ 個，合計 $O(n \cdot m)$；
- 驗證 F 約束時，每個 `'F'` 位置同樣掃描 $O(m)$ 的窗口，最多 $n$ 個，合計 $O(n \cdot m)$；
- 最終轉換字串需 $O(n + m)$。
- 總時間複雜度為 $O(n \cdot m)$。

> $O(n \cdot m)$

## 空間複雜度

- 輸出陣列與鎖定標記陣列各占 $O(n + m)$；
- `str2` 字元碼陣列占 $O(m)$。
- 總空間複雜度為 $O(n + m)$。

> $O(n + m)$
