# 3513. Number of Unique XOR Triplets I

You are given an integer array `nums` of length `n`, 
where `nums` is a permutation of the numbers in the range `[1, n]`.

A XOR triplet is defined as the XOR of three elements `nums[i] XOR nums[j] XOR nums[k]` where `i <= j <= k`.

Return the number of unique XOR triplet values from all possible triplets `(i, j, k)`.

**Constraints:**

- `1 <= n == nums.length <= 10^5`
- `1 <= nums[i] <= n`
- `nums` is a permutation of integers from `1` to `n`.

## 基礎思路

本題給定一個長度為 $n$ 的排列，其元素恰好覆蓋 $[1, n]$ 中的所有整數，要求統計所有滿足 $i \le j \le k$ 的三元組所能產生的相異互斥或值數量。由於 $n$ 可達 $10^5$，直接列舉所有三元組顯然不可行，必須從數值結構本身尋找規律。

在思考解法時，可掌握以下核心觀察：

- **索引允許重複，實際上等價於任意可重複選取三個值**：
  由於條件僅要求索引非嚴格遞增，同一個位置可以被重複選取，因此問題本質是「從 $[1, n]$ 中可重複挑選三個數字做互斥或」。

- **元素個數過少時無法形成有效組合**：
  當長度小於三時，能夠取得的互斥或結果僅限於原始數值本身，數量即等於陣列長度。

- **互斥或的結果被最高位元寬度所限制**：
  所有數值都不超過 $n$，因此互斥或後的位元寬度不會超過表示 $n$ 所需的位元數，可能結果必定落在某個以二為底的完整區間內。

- **當可選數值涵蓋足夠廣時，該區間會被完整填滿**：
  由於排列包含 $[1, n]$ 的全部數字，三個數字的互斥或組合具備足夠自由度，能夠覆蓋該位元寬度下的所有可能取值，包含零在內。

依據以上特性，可以採用以下策略：

- **先處理長度過小的邊界情況，直接回傳陣列長度**。
- **計算表示長度所需的位元數，並以此推得完整互斥或值域的大小**。
- **直接回傳該值域大小，無須任何實際列舉**。

此策略將原本組合爆炸的列舉問題化簡為常數時間的位元運算。

## 解題步驟

### Step 1：取得陣列長度

先取出陣列長度作為後續判斷與位元計算的依據。

```typescript
const length = nums.length;
```

### Step 2：處理元素個數不足的邊界情況

當長度小於三時，無法藉由組合產生新的互斥或值，可取得的結果僅有原始數值本身，因此數量即為陣列長度。

```typescript
// 當元素少於三個時，僅有原本的 n 個數值可被取得。
if (length < 3) {
  return length;
}
```

### Step 3：計算完整互斥或值域的大小並回傳

利用前導零計數求出表示長度所需的位元數，再取該位元寬度下的完整區間大小，即為所有相異互斥或結果的數量。

```typescript
// 嚴格大於 n 的最小二次冪，即完整互斥或值域的大小。
return 1 << (32 - Math.clz32(length));
```

## 時間複雜度

- 僅進行常數次的比較與位元運算；
- 未對陣列進行任何走訪。
- 總時間複雜度為 $O(1)$。

> $O(1)$

## 空間複雜度

- 僅使用固定數量的區域變數；
- 無任何額外配置的資料結構。
- 總空間複雜度為 $O(1)$。

> $O(1)$
