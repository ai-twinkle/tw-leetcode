# 1009. Complement of Base 10 Integer

The complement of an integer is the integer you get when you flip all the `0`'s to `1`'s 
and all the `1`'s to `0`'s in its binary representation.

For example, The integer `5` is `"101"` in binary and its complement is `"010"` which is the integer 2.
Given an integer `n`, return its complement.

**Constraints:**

- `0 <= n < 10^9`

## 基礎思路

本題要求找出一個整數的補數，也就是將其二進位表示中的每一個 `0` 與 `1` 全部翻轉後，所得到的新整數。需特別注意的是，翻轉只應作用在原本有效的二進位位數範圍內，不能將前方不存在的高位補上並一起翻轉。

在思考解法時，可掌握以下核心觀察：

* **補數運算的本質是逐位翻轉**：
  若某個二進位位數為 `1`，翻轉後會變成 `0`；若為 `0`，翻轉後會變成 `1`。

* **翻轉範圍必須與原數的二進位長度一致**：
  題目中的補數定義只針對該整數實際使用到的位數，因此需要先確定它的二進位長度，才能正確建立翻轉範圍。

* **可先構造一個同長度的全 1 結構**：
  若一個數的二進位長度為 `k`，則可以建立一個值為 `2^k - 1` 的整數；它的二進位形式會恰好是 `k` 個連續的 `1`。

* **補數可透過「全 1 結構減去原數」得到**：
  在同一個位數範圍內，將原數從全 1 結構中扣除，效果等同於把每一位進行翻轉。

依據以上特性，可以採用以下策略：

* **先取得原數的二進位位數長度**。
* **根據這個長度建立一個對應的全 1 遮罩值**。
* **用該遮罩值減去原數，直接得到翻轉後的結果**。

此策略能直接利用二進位結構的特性，在不需要逐位模擬的情況下完成補數計算。

## 解題步驟

### Step 1：建立同樣位數的全 1 遮罩並扣除原數

先取得輸入值的二進位長度，再建立一個該長度的全 1 整數；
最後用這個全 1 遮罩減去原數，即可得到所有位元翻轉後的補數結果。

```typescript
// 建立一個與 n 的二進位長度相同、且每一位皆為 1 的遮罩，再減去 n 以完成位元翻轉。
return Math.pow(2, n.toString(2).length) - 1 - n;
```

## 時間複雜度

- 需先將整數轉成二進位字串，以取得其位數長度；
- 其餘次方、減法等操作皆為常數時間；
- 若以二進位位數為 $k$，總時間複雜度為 $O(k)$。

> $O(k)$

## 空間複雜度

- 額外使用了一個二進位字串來表示原數；
- 該字串長度與原數的二進位位數成正比；
- 總空間複雜度為 $O(k)$。

> $O(k)$
